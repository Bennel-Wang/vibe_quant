# 统一量化数据源系统 使用说明

## 概述

本系统封装了多个中国股票市场量化 API（Tushare、Baostock、Easyquotation、Pytdx、Mootdx），提供统一的输入输出接口。系统自动在数据源之间降级切换，支持增量更新，并使用 NumPy（或可选 Cython）加速计算 23 项技术指标。

## 支持范围

| 类型 | 示例代码 | Tushare | Baostock | Easyquotation | Pytdx | Mootdx |
|------|---------|---------|----------|---------------|-------|--------|
| A股大盘 | 000001.SH | ✅ | ✅ | ✅(实时) | ✅ | ✅ |
| A股板块 | 399006.SZ | ✅ | ✅ | ✅(实时) | ✅ | ✅ |
| A股个股 | 000001.SZ | ✅ | ✅ | ✅(实时) | ✅ | ✅ |
| 港股大盘 | HSI.HK | ✅(index_global) | ❌ | ✅(FixedHKQuote) | ✅(ext market=27) | ❌ |
| 港股个股 | 00700.HK | ⚠️(需高积分) | ❌ | ✅(daykline) | ✅(ext market=31) | ❌ |

> **注意**: Tushare 港股个股 (`hk_daily`) 需要较高积分权限（≥2000积分）。Mootdx 的 `market='hk'` 返回的不是真实港股数据，`market='ext'` 官方已标注失效。

## 快速开始

### 命令行使用

```bash
# 获取单只股票数据（默认最近60天）
python main.py 000001.SZ

# 指定日期范围
python main.py 000001.SZ --start 20260101 --end 20260321

# 获取港股数据
python main.py 00700.HK --start 20260101

# 获取港股指数
python main.py HSI.HK --start 20260101
```

### Python API 使用

```python
from data_manager import DataManager

dm = DataManager()

# 获取数据（自动选择最优数据源，含降级逻辑）
df = dm.fetch("000001.SZ", start_date="20260101", end_date="20260321")

# 返回的 DataFrame 包含 OHLCV + 23项技术指标
print(df.columns.tolist())
# ['uniformed_stock_code', 'trade_date', 'open', 'high', 'low', 'close', 'vol',
#  'change', 'pct_chg', 'amount', 'date', 'volume',
#  'rsi_6', 'rsi_12', 'rsi_24', 'ma_5', 'ma_20', 'ma_60',
#  'macd', 'macd_signal', 'macd_histogram',
#  'boll_middle', 'boll_upper', 'boll_lower',
#  'kdj_k', 'kdj_d', 'kdj_j', 'volatility',
#  'volume_ma_5', 'volume_ma_20', 'volume_ratio', 'wr_14']
```

| 类型 | 格式 | 示例 |

| A股个股(沪) | `{6位代码}.SH` | `600000.SH`（浦发银行） |

## 数据源降级逻辑

### 历史数据获取流程

```
Tushare → Baostock（失败或无数据时自动切换）
```

### 实时数据获取流程

```
Easyquotation → Pytdx → Mootdx（失败或无数据时自动切换）
```

### 降级触发条件

1. **数据缺失**: 数据源返回 `None` 或空 DataFrame
2. **数据异常**: 存在 0 值、NaN 值、high < low 等无效数据
3. **非最新数据**: 当前为交易日，但获取的最新数据为上一个交易日的历史数据 → 自动切换到实时数据源

### 增量更新

- 本地已有有效数据的日期不会被重新下载
- 只更新缺失日期或覆盖无效数据
- 减少 API 调用次数，避免频率限制

## 交易日历

系统内置 A股和港股交易日历（2024-2026年），用于判断：
- 当前是否为交易日
- 数据是否需要更新到最新
- 哪些日期应该有数据

### A股与港股差异

- A股交易日参考上交所/深交所
- 港股交易日参考港交所
- 两市的节假日安排不同，系统独立处理

## 技术指标计算

系统基于 OHLCV 基础数据自动计算以下 23 项技术指标：

- **RSI**: 6日、12日、24日相对强弱指标
- **MA**: 5日、20日、60日移动平均线
- **MACD**: DIF线、DEA线（signal）、柱状图（histogram）
- **BOLL**: 布林带中轨、上轨、下轨（20日，2倍标准差）
- **KDJ**: K值、D值、J值（9日周期）
- **波动率**: 20日收益率标准差
- **成交量指标**: 5日均量、20日均量、量比
- **WR**: 14日威廉指标

### Cython 加速（可选）

如果安装了 MSVC 编译器，可以编译 Cython 版本以提高计算性能：

```bash
cd indicators
python setup.py build_ext --inplace
```

如果编译失败，系统自动使用纯 NumPy 实现（功能完全相同，速度稍慢）。

## 各数据源详细说明

### Tushare

- **类型**: 历史数据（优先）
- **A股**: `pro.daily()` / `pro.index_daily()`
- **港股指数**: `pro.index_global(ts_code='HSI')` 等国际指数接口
- **港股个股**: `pro.hk_daily()`（需较高积分）
- **频率限制**: 每分钟有调用次数限制，积分越高限制越宽
- **注册**: https://tushare.pro/register

### Baostock

- **类型**: 历史数据
- **支持**: 仅A股（深沪两市股票、指数）
- **特点**: 免费、无积分限制
- **代码格式**: `sh.600000` / `sz.000001`
- **注意**: 成交量单位为"股"，系统自动转换为"手"

### Easyquotation

- **类型**: 实时数据 + 港股日K历史
- **A股实时**: 新浪财经接口 (`use('sina')`)
- **港股日K**: `use('daykline')` 获取港股历史日K线
- **港股实时**: `use('hkquote')` / 修复版 `FixedHKQuote`（支持 HSI 等字母代码）
- **特点**: 实时性强，无需注册

### Pytdx

- **类型**: 实时数据（TDX协议直连行情服务器）
- **A股**: 标准行情接口（market: 0=深圳, 1=上海）
- **港股**: 扩展行情接口（ext）— market=27 为港股指数, market=31 为港股个股
- **扩展服务器**: `180.153.18.176:7721`（已测试可用）
- **注意**: 港股数据通过 ext 扩展服务器获取，实时性可能有滞后

### Mootdx

- **类型**: 实时数据（基于 pytdx 封装）
- **支持**: 仅A股（深沪两市股票、指数）
- **特点**: 简单易用，`Quotes.factory(market='std')`
- **注意**: `market='hk'` 返回的不是真实港股数据；`market='ext'` 扩展市场接口已失效

## 常见问题

### Q: Tushare 港股个股获取失败？
A: Tushare 的 `hk_daily` 接口需要较高积分权限（≥2000积分）。可以使用 Easyquotation 的 `daykline` 或 Pytdx ext 作为替代。

### Q: Pytdx 港股连接超时？
A: 检查 `config.py` 中的 `PYTDX_EX_HOSTS` 配置，确保使用可用的扩展行情服务器。推荐 `180.153.18.176:7721`。

### Q: Cython 编译失败？
A: 需要安装 Microsoft Visual C++ Build Tools (MSVC)。如果无法安装，系统会自动使用纯 NumPy 实现，功能完全相同。

### Q: 为什么 Mootdx 不支持港股？
A: Mootdx 的 `market='hk'` 实际使用标准 TDX 协议，返回的不是真实港股行情；`market='ext'` 扩展市场接口已官方标注失效。建议使用 Pytdx ext 或 Easyquotation 获取港股数据。

### Q: 数据有重复日期怎么办？
A: 系统的增量更新逻辑会自动去重，以最新获取的数据为准。
