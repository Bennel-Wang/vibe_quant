# 主力成本与筹码分布估算方法

## 一、背景与概念

**筹码（Chip）**：每手股票被持有的成本价集合。当持有者以某价格买入后，该价格区间就积累了一定数量的"筹码"。随着日常交易换手，筹码在不同价格区间之间流动。

**主力成本**：大资金（主力）建仓/持仓的平均价格。通过分析换手率和成交量加权可以估算。

---

## 二、筹码分布模型（换手率累积法）

### 2.1 原理

将历史每日 OHLCV 数据拆解为"筹码流动"：

1. **当日换手**：当日成交量占总流通股本的比例 = 换手率 `t = V_day / V_float`
2. **旧筹码减少**：所有价格区间的筹码乘以 `(1 - t)`，模拟旧持有者卖出
3. **新筹码分配**：当日新买入的筹码 `V_day * t` 按当日的三角分布分散到 `[low, high]` 区间
4. **三角分布权重**：以当日均价 `mid = (high + low) / 2` 为峰值，向两端线性递减：
   ```
   w(p) = max(0, 1 - |p - mid| / ((high - low) / 2))
   ```

### 2.2 计算步骤

```
初始化: chip_dist[0..N-1] = 0  (N 个价格区间)

for each 交易日 i:
    t = volume[i] / total_float_volume  # 当日换手率（近似）
    chip_dist *= (1 - t)               # 旧筹码减少
    for each 价格区间 j in [low_i, high_i]:
        chip_dist[j] += volume[i] * t * w(price_j)  # 新筹码分配

归一化: chip_dist /= chip_dist.sum()  # 转为百分比
```

### 2.3 关键参数
| 参数 | 典型值 | 说明 |
|------|-------|------|
| 价格区间数 N | 60 | 在历史最高/最低价之间等分 |
| 历史回看窗口 | 250 交易日 | 约 1 年 |
| 总流通量 | 近 250 日成交量之和（近似） | 真实数据需流通股本 |

### 2.4 输出指标

- **筹码分布图**：各价格区间的筹码占比，横轴为价格，纵轴为持仓占比
- **获利盘比例**（Profit Ratio）：当前价格以下的筹码占比，越高说明更多人持仓浮盈
- **主力成本区间**：筹码密度最高的区间（最高峰 ±2 个区间）

## 三、成本均线法（VWAP 法）

### 3.1 原理

使用**成交量加权平均价格（VWAP）**，近似估算过去 N 天内买入持仓者的平均成本：

```
VWAP_N = Σ(典型价格 × 成交量, 过去N天) / Σ(成交量, 过去N天)

典型价格 = (High + Low + Close) / 3
```

### 3.2 常用周期
| 指标名 | 窗口 | 含义 |
|--------|------|------|
| `major_cost_20` | 20 日 | 近 1 个月均成本（短线主力参考） |
| `major_cost_60` | 60 日 | 近 3 个月均成本（中线主力参考） |
| `major_cost_120` | 120 日 | 近 6 个月均成本（半年线主力参考） |
| `major_cost_turnover` | 60 日 | 基于换手率加权的综合成本 |

### 3.3 解读方式

- **股价 > VWAP**：当前价格高于过去 N 天的平均持仓成本，大多数持有者处于盈利状态
- **股价 ≈ VWAP**：价格运行在成本支撑/压力区，重要参考位
- **股价 < VWAP**：大多数持有者处于亏损状态，此区域可能存在较强支撑（被套盘等解套意愿弱化抛压）

## 四、价格-成交量分布（PVD）

将历史交易日按收盘价区间分组，统计每个价格区间内的**总成交量**和**总换手次数**：

```
PVD[price_bucket] = Σ volume_i  for each i where price_i ∈ price_bucket
```

高成交量区间 → 历史大量筹码在此换手 → 未来该区间对价格有支撑/压力作用。

---

## 五、主力成本与普通成本的区别

| 方法 | 适用场景 | 局限性 |
|------|---------|--------|
| 简单 VWAP | 全市场平均持仓成本 | 无法区分主力与散户 |
| 换手率筛选 | 低换手期（主力不动）= 筹码稳定区 | 需要流通股本数据 |
| 大单追踪 | Level-2 数据中的大单方向分析 | 需要 Tick 数据 |
| 筹码分布 | 直观显示全市场筹码位置 | 总流通量近似，存在误差 |

> **注意**：以上估算方法均为近似模型，基于公开的 OHLCV 数据推导。真实主力成本受到大宗交易、场外定增、配股等因素影响，无法从日线数据中精确还原。实际操作中应结合多种指标综合判断。

## 六、在本系统中的实现

本系统在 `quant_system/indicators.py` 的 `TechnicalIndicators` 类中实现了以下方法：

- **`calculate_major_cost(df)`**：计算 20/60/120 日 VWAP 成本均线
- **`calculate_chip_distribution(df, n_bins=60)`**：使用换手率累积模型计算筹码分布

可通过 API 接口 `/api/stock/<code>/chip_distribution` 获取实时筹码分布数据。
