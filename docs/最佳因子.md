# 最佳因子页面 — 实现计划

## 问题陈述
在"因子挖掘"页面（`/factors`）的现有三个 Tab（因子筛选 / 估值分析 / 因子优化）旁边，新增第四个 Tab **「最佳因子」**。

该功能对**批量股票**（可用户选择）遍历**所有策略**，通过5年回测数据综合评分，筛出每只股票最适合的量化策略，并给出**加仓建议**。

---

## 核心设计

### 1. 综合评分体系

对每只股票的每条策略，基于以下三个指标计算**综合得分（0~100分）**：

| 指标 | 权重 | 说明 |
|------|------|------|
| 年化收益率 (annual_return) | 40% | 超过基准 6% 后按比例加分，上限参考 50% |
| 胜率 (win_rate) | 30% | 线性映射，50%→0分，100%→100分 |
| 最大回撤 (max_drawdown_pct) | 30% | 越低越好，0%→100分，40%→0分 |

**综合得分公式：**
```
annual_score   = clamp((annual_return - 6) / (50 - 6), 0, 1) * 100
win_score      = clamp((win_rate - 50) / 50, 0, 1) * 100
drawdown_score = clamp((40 - max_drawdown_pct) / 40, 0, 1) * 100

composite_score = annual_score * 0.4 + win_score * 0.3 + drawdown_score * 0.3
```

### 2. 准入门槛（硬性过滤）

任意一条不满足则该策略直接淘汰（不进入评分）：

- 胜率 ≥ 50%
- 最大回撤 ≤ 40%
- 年化收益 ≥ 6%
- 总交易次数 ≥ 3（排除极少交易的无效策略）

若所有策略均不达标 → 返回 `"该股票不适合量化"`

### 3. 加仓建议算法

以最优策略的**历史最大回撤**作为仓位上限依据：

```
max_position_pct = min(max_drawdown_pct, 30)  # 最高仓位不超过最大回撤幅度（上限30%）
step = max_position_pct / 3                    # 分三次加仓
```

**举例（最大回撤30%）：**
- 初始建仓：10%
- 下跌 10% 时加仓至 20%
- 下跌 20% 时加仓至 30%（满仓）
- 加仓建议我希望看到的是具体几个点加一次，一次加多少仓位这种更具体的 （仓位上限应该根据策略的最大仓位定，而不光是根据最大回撤来定，比如策略最大仓位上限是60%，每次操作的仓位比例是20%，那意味着加仓3次，回撤最大是30%，那可以每次跌10%加仓一次，每次加仓20%。如果是混合策略就取小的那个仓位）

### 4. 回测时间周期

固定 **5年**（从今天往前推5年），不允许用户修改，保证横向可比性。

## 实现范围

### 后端（`web_app.py`）
1. **新 API：`POST /api/factors/best`**
   - 接收：`codes`（股票代码列表，为空则取全部持仓股票）、`initial_capital`
   - 自动计算5年日期区间
   - 对每只股票：预计算指标 → 遍历所有策略对 + _generate_factor_strategies() 参数化策略 → 综合评分 → 筛选 → 返回最优
   - 返回：每只股票的最优策略、综合得分、加仓建议；或不适合量化标记

2. **进度推送：`GET /api/factors/best/progress`**
   - 轮询接口，返回当前处理进度（已完成/总数/当前股票）

### 前端（`factors.html`）
3. **新增第四个 Tab：「最佳因子」**
   - 左侧控制面板：股票选择（全部/自选）、初始资金、启动按钮
   - 右侧结果区：
     - 进度条（处理中实时更新）
     - 汇总表格：股票 / 股票名字/最优策略 / 综合得分 / 年化% / 胜率% / 最大回撤% / 加仓建议
     - 点击行可展开：该股票所有有效策略对比 + 评分雷达图
     - 不适合量化的股票单独列出

### 5. 跨策略混搭优化（Cross-Combination）

在正常评分完成后，对**得分接近的前几名策略**尝试"买入+卖出"交叉组合：

**触发条件：**
- 通过门槛的策略 ≥ 2 条
- 第1名与第2名综合得分差值 ≤ 15分（认为"得分相近"）

**组合方式（仅限2次额外尝试）：**
1. 第1名的买入规则 + 第2名的卖出规则 → 组成新策略对，回测评分
2. 第2名的买入规则 + 第1名的卖出规则 → 组成新策略对，回测评分

**纳入评分池**，与原策略统一排名，取最终最优。

**设计原则：**
- 不进行全量排列组合，只对"近似得分"的头部两名做交叉
- 混搭策略标注来源（如 "策略A买入 × 策略B卖出"），方便用户理解
- 策略不是一定用混搭，一定要是混搭效果更好才用，并且混搭策略不能违背准入门槛
