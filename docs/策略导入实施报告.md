## 三、系统修复：策略热加载

**问题**：Flask 应用启动时一次性加载 strategies.json 到内存，之后文件修改不会反映在 API 响应中。

**修复方案**：在 StrategyManager 添加 `reload_from_file()` 方法，基于文件 mtime 检测变化，并在以下 API 路由头部调用：
- GET /api/strategies
- GET /api/strategy/<name>
- POST /api/backtest/run

**代码位置**：
- `quant_system/strategy.py`：reload_from_file() 方法（约第1062行）
- `quant_system/web_app.py`：三处路由调用

### 7.3 最优策略（V19）详解

**适用场景：** A股动量型标的（新能源、半导体、化工龙头），大盘中性偏强时使用。

#### 买入逻辑（三通道）

| 通道 | 条件 | 仓位 | 逻辑 |
|---|---|---|---|
| CH1 主力通道 | MACD金叉 + RSI(30-62) + 站20日线 + 量比>1.4 + 10日不弱大盘 | 20% | 高置信度趋势买入 |
| CH2 周线超跌 | 周线RSI<40分位 + 日线回升 + 布林中轨区 | 10% | 超跌修复买入 |
| CH3 趋势延续 | 站20/60双线 + 20日跑赢大盘3% + MACD正区 | 5% | 加仓顺势 |

#### 卖出逻辑（延迟止盈）

| 条件 | 操作 | 目的 |
|---|---|---|
| 周线RSI>80分位 + 日线RSI>70 | 卖出50% | 高位首次止盈，保留仓位继续跑 |
| 周线RSI>88分位 | 卖出60% | 历史极高位，防止顶部逆转 |
| 跌破20日线×0.97 | 清仓100% | 硬止损，控制风险 |
| MACD死叉 + 跌破20日线 | 卖出60% | 趋势反转信号 |
| RSI>85 + 当日下跌 | 卖出50% | 高位转跌 |

**关键设计决策：** 将止盈阈值从V10的周线RSI>75提高到>80，实现"让利润更充分奔跑"。  
对比V10/V16(年化2.9-3.1%)，V19年化3.6%，在002594(比亚迪)+53.4%、300750(宁德时代)+52.6%的大行情中完整捕获。

#### 排除规则

- RSI > 72（不追高）  
- 大盘极度恐慌 overall_score < -50  
- 10日持续跑输大盘 rel_strength_10 < -8

### 7.4 关键优化洞察

1. **Sharpe 0.63 是该策略族的局部最优**：V10/V16/V19 三个版本 Sharpe 均为 0.63，说明已到达当前指标组合的上限。  
2. **提高止盈阈值比降低止损更有效**：让赢家跑更久（RSI 75→80）比频繁止损对 Sharpe 的贡献更大。  
3. **长江电力(600900)是策略短板**：作为慢牛型公用事业股，MACD 动量策略胜率仅 40-44%。若针对不同类型标的使用不同策略，整体表现会更好。  
4. **仓位优化有边际收益**：V16 将 Rule1 仓位从 15%→20%，年化从 2.9%→3.1%，简单调仓即可提升。  
5. **过复杂的条件反而更差**：V20（4通道+7卖出条件）Sharpe 仅 0.09，V18（加布林底部规则）Sharpe 0.32，说明奥卡姆剃刀适用于量化策略。
